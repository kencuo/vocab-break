# 背词小窗（Vocab Break）

SillyTavern 第三方扩展：在 AI 生成时提供“背词/阅读”小窗，并附带**阅读对话**功能。背词与阅读为**两个独立面板**，且在设置里**二选一启用**，不会同时出现在屏幕上。

## 主要功能

- **背词面板**：展示单词与释义，支持标记“不认识/模糊/熟悉/清除”。
- **阅读面板**：导入本地 TXT 分页阅读，支持字体大小与行距调整。
- **阅读对话**：阅读小窗的对话浮窗（可收起/展开），支持 OpenAI 兼容接口与流式回复。
- **远程阅读**：从 GitHub raw 目录解析章节并加载阅读内容。
- **收缩栏**：面板默认收缩，点击收缩栏可展开；可拖拽移动位置。
- **权重循环**：不熟悉词条出现概率更高。
- **主题与自定义 CSS**：简约/档案/古风/自定义 CSS。
- **面板尺寸**：可在设置中独立调整背词/阅读面板的宽高。
- **本地缓存**：词库、状态与阅读内容保存至浏览器本地（容量超限会提示）。
- **人设/世界书注入**：阅读对话会自动注入用户与角色人设，并可检索当前世界书（Lorebook）。

## 内置词库

- 考研词汇（Apache-2.0）
- NETEM 词汇（CC BY-NC-SA）
- 背单词词库（GPL-3.0）
- CET-4 / CET-6 词库（KyleBing/english-vocabulary，TXT 与 JSON 均可用，**许可未知**）

## 安装

1. 将扩展目录放入：
   `SillyTavern-release/public/scripts/extensions/third-party/vocab-break`
2. 进入 SillyTavern → Extensions，启用“背词小窗”。

## 使用说明

### 启用与开关（重要）
- 设置里提供“**启用背词小窗**”与“**启用阅读小窗**”。
- 两者**互斥**：启用一个会自动关闭另一个，避免同时显示。
- 若需要“都关闭”，可把两个开关都关掉。

### 背词面板
1. 选择词库来源（或填写自定义链接）。
2. 点击“刷新词库”加载词条。
3. 面板中可：
   - 显示/隐藏释义
   - 跳过
   - 标记“不认识/模糊/熟悉/清除”

### 阅读面板
1. 选择编码（自动识别/UTF-8/GBK/Big5/UTF-16），点击“导入 TXT”加载本地文本。
2. 使用“上一页/下一页”翻页。
3. 在设置中调整分页大小、字体大小与行距；可开启“翻页保持位置”避免自动回到页首。

### 阅读对话（AI）
1. 在阅读面板右上角点击“对话”，打开阅读对话浮窗。
2. 在扩展设置中填写 **API Provider/Endpoint/Key/Model**（OpenAI 兼容接口）。
3. 输入消息 → 点击“发送” → 点击“生成回复”。
4. 支持流式回复（可在设置里开关）。

对话上下文包含：
- 当前阅读历史（可通过“阅读历史注入”开关控制）
- 用户/角色人设（来自当前 Persona 与角色卡）
- 当前世界书（Lorebook）匹配结果
- 最近对话记录

### 远程阅读
1. 在阅读面板的“远程教材”区域选择来源。
2. 点击“加载目录”解析章节。
3. 选择章节后点击“加载章节”。

### 收缩栏与拖拽
- 面板默认收缩，关闭后会显示对应收缩栏。
- 点击收缩栏展开面板。
- 拖动面板标题栏或收缩栏可移动位置。

### 自动弹出
- “生成时自动弹出/结束自动关闭”只对**背词面板**生效。
- 阅读面板不会在生成时自动弹出。

## 词库格式支持
支持以下格式：
- JSON 数组：`["word", "word2"]`
- JSON 对象：`{ data: [...], list: [...], items: [...] }`
- 文本行：
  - `word<TAB>meaning`
  - `word - meaning`
  - `word|meaning`
- 词条字段自动匹配：
  - 单词：`word/en/english/term/key/headword/spelling`
  - 释义：`translation/meaning/cn/zh/trans/definition/explain`

## 本地缓存与书架
- 词库与标记状态按 URL 缓存到浏览器本地。
- 阅读内容会缓存到本地；超出上限将只在本次载入有效。
- 阅读面板带**本地书架**：导入的 TXT 会自动入库，支持打开/删除。
- 本地书架会为**每本书保存阅读进度**（页码与滚动位置）。

## 世界书（Lorebook）说明
- 阅读对话会触发一次世界书检索，并将匹配内容注入对话提示词。
- 使用的是“dry run”模式：**不会**修改世界书的计时/冷却效果（sticky/cooldown），只用于当次回复。
- 触发关键词来源：阅读历史 + 最近对话 + 人设/角色信息。

## 注意事项
- 阅读对话依赖 `generateRaw`，旧版本或不支持的环境会提示不可用。
- 破限词目前仅作为系统级提示词保留，阅读对话默认以角色身份回复。
